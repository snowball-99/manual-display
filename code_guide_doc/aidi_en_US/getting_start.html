Script Editor Getting Started Guide#

Function template#

Function template function description#

The AIDI Script Editor offers commonly used auxiliary functions that can be directly inserted into scripts as function templates. It also provides a function template management feature, allowing users to encapsulate frequently used functions as reusable templates during script editing. For details on the built-in function templates, please refer to the Instructions for Built in Function Template

How to view existing Function templates#

Click the “Function Template Management” button in the upper-right corner to open the function management template interface. The list on the left displays the existing template functions.

Custom Function Template#

  1. Script Editor - Function template management in the upper right corner

  2. New - Enter function alias - Enter function description - Enter edited function content - Click register

  3. Registration check: Ensure functions have no duplicate names, no single functions, and no other unknown non-runnable reasons. If the requirements are not met, modify and register the functions (to temporarily check the next step through registration, use Ctrl+Z to restore the initial state or comment out the current function contents).

Batch Export - Batch Import Function Template#

AIDI supports exporting function templates in bulk and then importing them on other machines to enable multi-person sharing of function templates.

  1. Script Editor - Function template management in the upper right corner

  2. Select the “Export” label next to “Function List”. Choose the export path, then complete the batch export

  3. At the comprehensive judgment node of any AIDI project that requires importing, select the “Function Import” label next to the “Function List”. Then, batch select the address where the function is located and complete the batch import. The function import process will perform function alias and name deduplication

How to Insert Function Template into Script Editing Area#

Select the code line for inserting the Function template in the script editing area, and enter templ.. The script editor will list all Function templates, then select the desired template to insert it directly.

User variables#

User variables function description#

To facilitate modifying script variables during secondary development, it’s recommended to manage all variables that may need setting using User variables instead of directly writing them into the script code. The current User variables support Number, String, and Bool types.

How to set User variables#

Click on Add User variables in the User variables settings section on the left and set the value.

How to use User variables in scripts#

User variables are stored in the global variable VFLOW_USER_VARS, which is a Python dictionary. To access a user variable named var0, you can use VFLOW_USER_VARS[“var0”] directly in the script.

Convenient debugging#

The AIDI scripts support printing variables directly to the log area using the print function. You can view the script log in the bottom left corner.
